阿里云+lnmp+wordpress配置博客

发表于2022-12-29|更新于2022-12-29
|阅读量:

环境

LNMP（linux+nginx+mysql+php）

版本

服务器：阿里云
linux：ubuntu 20.04
nginx：1.18.0 (Ubuntu)
mysql：8.0.28-0ubuntu0.20.04.3 for Linux on x86_64 ((Ubuntu))
php：7.4.3 (cli)

步骤

  参考教程阿里云官方文档及部分CSDN、简书、博客园等 安装前现在阿里云安全策略中打开http80端口，否则无法打开网页


1. 更换操作系统后更新源 `apt update && apt upgrade -y && apt-get update && apt-get upgrade -y` 2. 安装mysql `apt install mysql-server -y` 3. 安装php及相关依赖项 `apt install php php-mysql php-curl php-dom php-exif php-fileinfo php-fpm php-gd php-hash php-json php-mbstring php-mysqli php-openssl php-pcre php-xml libsodium -y` 其中php-fpm和php-mysql必须安装（php-fpm安装后可能会顺便安装apache2，若要安装nginx则必须停用apache，否则会端口冲突，`ps -ef|grep apache`查看进程后`kill -9 pid`强制结束进程） 4. 安装nginx `apt install nginx -y` 5. 配置mysql 1. mysql设置启动密码 `mysql_secure_installation` 2. 按Y进行配置 选择密码策略强度，0表示弱，2表示强 3. 输入新密码 4.按Y确定使用该密码 5. 按Y删除匿名账户 6.确定是否需要远程连接，按Y不需要远程连接，非Y需要连接 7. 按Y删除测试库和访问权限 8. 按Y重新加载授权表 9.使用新密码登录数据库 `mysql -u root -p` 10. 新建数据库（如命名为wordpress） `create database wordpress;` 11. 新建用户 `create user '用户名'@'localhost' identified by '用户密码';` 12. 赋予用户对数据库wordpress的全部权限 `grant all privileges on wordpress.* to '用户名'@'localhost';` 13. 使配置生效 `flush privileges;` 14. \q或exit;退出 6. 确认nginx和php工作情况 1. 在浏览器输入服务器ip，若能出现nginx默认界面则nginx工作正常 2. 在默认目录中（一般为/var/www/html，若不是则在/etc/nginx/sites-available/default文件中找root一行）新建phpinfo.php文件并输入保存 3. 访问ip/phpinfo.php，若能打开phpinfo界面则php运行正常（建议测试后删除该文件防止泄露隐私信息） 7. 下载[wordpress](https://cn.wordpress.org/latest-zh_CN.tar.gz "wordpress")并解压复制至网页根目录，同时在/etc/nginx/sites-available/default中把root路径改为网页根目录/wordpress 8. 将WordPress安装目录下的wp-config-sample.php文件复制到wp-config.php文件中，并将wp-config-sample.php文件作为备份 `cp wp-config-sample.php wp-config.php` 9. 编辑wp-config.php文件，WordPress网站的数据信息将通过数据库的用户保存在名为wordpress的数据库中 保存退出 10. 在wordpress根目录中命令 `chown 用户名:用户组 wordpress路径/wp-content` 赋予上传目标文件夹的上传权限，用户名和用户名组在nginx.conf中查看 10. 使用`whereis php.ini`搜索php.ini文件（一般在/etc/php/7.4/fpm中），找到`upload_max_filesize`一行，将2M改为较大的任意数值，取消上传大小限制 10. 重启nginx和php 11. 浏览器访问ip，不出意外的话即可进入wordpress登录界面，按步骤操作即可 12. 若仍无权限上传文件则在nginx.conf的server栏中添加`client_max_body_size 20M;`并重启后应该可以解决
